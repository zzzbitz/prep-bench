# Frontend Design

Build real, production-grade frontend code with a clear and memorable visual direction. Avoid generic layout and styling defaults.

## Define Direction First

Before coding, decide and state:

1. Purpose: identify user, use case, and task flow.
2. Tone: commit to one strong visual direction (for example editorial, brutalist, retro-futuristic, organic, or luxury).
3. Constraints: respect framework, accessibility, performance, and device requirements.
4. Signature move: choose one standout visual or interaction idea that makes the interface memorable.

Use this direction as a hard constraint for all implementation choices.

## Implement with Intent

Produce working HTML/CSS/JS or framework code that is:

1. Functional and shippable.
2. Cohesive across typography, spacing, color, and motion.
3. Accessible (semantic structure, keyboard support, sufficient contrast, reduced-motion handling where appropriate).
4. Responsive on desktop and mobile.

## Apply High-Quality Visual Rules

1. Typography: pick characterful type pairings; avoid generic defaults such as Arial, Roboto, Inter, and system-only stacks unless explicitly required.
2. Color: define CSS variables and commit to a dominant palette with deliberate accents.
3. Motion: prioritize a few meaningful moments (for example page-load sequencing, reveal choreography, and intentional hover/focus behavior).
4. Composition: use deliberate asymmetry, overlap, density, or whitespace based on the chosen direction.
5. Atmosphere: build depth with layered backgrounds, texture, pattern, light/shadow, or shape systems; avoid flat, context-free surfaces.

## Avoid Generic Patterns

Do not default to:

1. Purple-on-white gradient templates.
2. Cookie-cutter hero/features/testimonials blocks unless the user explicitly asks for them.
3. Repetitive component styling that ignores the selected aesthetic direction.

## Match Complexity to Vision

1. For maximal concepts, implement richer structure, layered visuals, and orchestrated animation.
2. For minimal concepts, implement restraint with precise spacing, rhythm, and subtle detail.

Keep design choices intentional and consistent from first paint to fine interaction details.
