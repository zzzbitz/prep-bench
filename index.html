About PrepBench

PrepBench

NL-Driven Data Preparation Benchmark

PrepBench evaluates LLM-based agents on natural-language-driven data preparation across three capabilities: disambiguation, prep-code generation, and code-to-workflow abstraction.

Three Evaluation Capabilities

(1)

Interactive Disambiguation

Tests whether systems can detect and resolve ambiguous intents through interaction.

(2)

Prep-Code Generation

Tests whether systems can generate correct preparation code from user requests.

(3)

Code-to-Workflow Translation

Tests whether generated code can be mapped to GUI workflows.

Task Example

Tour de France Team Analysis

RAW INPUT

Rank Rider Team Times Gap
1 GERAINT THOMAS TEAM SKY 83h 17' 13'' -
2 TOM DUMOULIN TEAM SUNWEB 83h 19' 04'' + 00h 01' 51''
3 CHRIS FROOME TEAM SKY 83h 19' 37'' + 00h 02' 24''
4 PRIMOŽ ROGLIC TEAM LOTTO NL 83h 20' 35'' + 00h 03' 22''

Filter to teams with 7+ finishing riders. Parse time gaps into minutes. Compute average gap per team. Keep only teams averaging ≤100 minutes behind the leader.

1. Disambiguation

Q₁: The Gap column contains "-" for some riders. Should I treat it as missing data?
A₁: No - "-" means the race leader. Their gap is zero.

Q₂: Should the team average gap be rounded or floored to an integer?
A₂: Floored. A team with 96.8 min should show 96.

2. Prep-Code Generation

python
# Leader's gap is "-", treat as zero
df["gap_min"] = np.where(
    df["Gap"] == "-", 0.0,
    df["Gap"].str.extract(r"(\d+)h\s*(\d+)'\s*(\d+)''")
      .astype(float).apply(
        lambda r: r[0]*60 + r[1] + r[2]/60, axis=1))
# Aggregate by team, floor the result
team = df.groupby("Team").agg(
    avg_gap=("gap_min", "mean"),
    riders=("Rider", "count"))
team["avg_gap"] = np.floor(team["avg_gap"])
